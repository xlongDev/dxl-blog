## 1. HTTP 的状态码有哪些？重点讲讲 304 和 403

### 原理剖析
HTTP 状态码是服务器对请求的响应，分为 5 类：1xx（信息）、2xx（成功）、3xx（重定向）、4xx（客户端错误）、5xx（服务器错误）。前端常碰到的有 200（成功）、404（未找到）、500（服务器错误），但 **304** 和 **403** 也很关键。

- **304 Not Modified**：资源没变，浏览器用缓存。配合 `ETag` 或 `If-Modified-Since`，服务器判断后返回 304，省流量。
- **403 Forbidden**：权限不够，服务器拒绝访问，比如没登录或跨域限制。

### 前端场景与最佳实践
304 是性能优化的好帮手，比如缓存 JS 文件。403 常见于 API 调用失败，建议前端加错误处理。

**最佳实践**：
1. 设置 `Cache-Control: max-age=3600` 和 `ETag`，让静态资源走 304。
2. 用 Axios 拦截器捕获 403，提示用户登录。

### 面试回答示范
“HTTP 状态码有 5 类，比如 200 是成功，404 是没找到。304 是 ‘Not Modified’，服务器发现资源没变，返回 304，让浏览器用缓存，适合优化静态资源。403 是 ‘Forbidden’，权限问题，可能没登录。前端会用拦截器处理 403，跳转登录页。”

## 2. GET 和 POST 的区别是什么？底层原理是什么？

### 原理剖析
GET 和 POST 是 HTTP 的两种方法，区别不只在用途，底层也有讲究。

- **GET**：拿数据，参数在 URL（`?id=1`），明文传输，大小有限（浏览器限制几 KB），幂等。
- **POST**：提交数据，参数在请求体（JSON、表单），更安全，无大小限制（看服务器）。

底层都是 TCP，但 GET 常被缓存，POST 不会。

### 前端场景与最佳实践
GET 适合查询列表，POST 适合表单提交。注意 GET 参数泄露风险。

**最佳实践**：
1. GET 请求加缓存头（如 `Cache-Control`）。
2. POST 敏感数据用 HTTPS。

### 面试回答示范
“GET 是获取数据，参数在 URL，适合查询，幂等；POST 是提交数据，参数在 body，适合表单。GET 明文不安全，POST 更适合敏感数据。前端用 GET 查列表，POST 提交表单，敏感请求走 HTTPS。”

## 3. TCP 和 UDP 的区别是什么？前端会用到 UDP 吗？

### 原理剖析
TCP 和 UDP 是传输层协议，设计理念不同。

- **TCP**：可靠，面向连接，三次握手、四次挥手，有拥塞控制。
- **UDP**：不可靠，无连接，速度快，适合实时场景。

### 前端场景与最佳实践
前端主要用 TCP（HTTP/HTTPS），但 WebRTC 用 UDP 做实时通信。

**最佳实践**：
1. 普通请求走 TCP。
2. 实时应用（如视频聊天）用 WebRTC，走 UDP。

### 面试回答示范
“TCP 可靠，三次握手建连接，适合 HTTP；UDP 快但不可靠，适合直播。前端主要用 TCP，但 WebRTC 用 UDP 做实时通信，比如视频会议。”

## 4. TCP 三次握手的过程是什么？为什么不是两次？

### 原理剖析
三次握手建立 TCP 连接：
1. 客户端发 SYN。
2. 服务器回 SYN+ACK。
3. 客户端回 ACK。

两次不行，因为无法确认双向通信能力，三次确保可靠。

### 前端场景与最佳实践
HTTPS 的 TLS 握手增加延迟，建议用 HTTP/2 复用连接。

### 面试回答示范
“TCP 三次握手是客户端发 SYN，服务器回 SYN+ACK，客户端再回 ACK，确保双方能通信。两次不行，可能一方收不到确认。前端 HTTPS 握手多，我会用 HTTP/2 优化。”

## 5. DNS 解析的过程是什么？有什么优化方法？

### 原理剖析
DNS 把域名解析成 IP：
1. 浏览器缓存。
2. 本地 DNS。
3. 根服务器 → 顶级域名服务器 → 权威服务器。

### 前端场景与最佳实践
DNS 影响首屏加载。优化：
1. `<link rel="dns-prefetch">` 预解析。
2. 用 CDN 缩短路径。

### 面试回答示范
“DNS 解析从浏览器缓存到本地 DNS，再到根、顶级、权威服务器。前端用 DNS 预解析或 CDN 加速加载。”

## 6. HTTPS 是怎么保证安全的？证书的作用是什么？

### 原理剖析
HTTPS 用 TLS/SSL：
1. 非对称加密交换密钥。
2. 对称加密传输数据。
3. 证书验证身份。

证书由 CA 签发，含公钥，防中间人攻击。

### 前端场景与最佳实践
所有 API 走 HTTPS，证书续期要及时。

### 面试回答示范
“HTTPS 用 TLS 加密和认证，非对称加密换密钥，对称加密传数据，证书验证身份。前端全用 HTTPS，证书过期前续期。”

## 7. CORS 是什么？怎么解决跨域问题？

### 原理剖析
CORS 是浏览器限制跨域，服务器用 `Access-Control-Allow-Origin` 放行。

### 前端场景与最佳实践
跨域常见于 API 调用。解决：
1. 后端配置 CORS。
2. 开发用代理。

### 面试回答示范
“CORS 是浏览器跨域限制，服务器设头放行。前端用代理或让后端配 CORS 解决。”

## 8. WebSocket 和 HTTP 的区别是什么？

### 原理剖析
HTTP 是单向请求-响应，WebSocket 是双向持久连接，通过 HTTP 握手建立。

### 前端场景与最佳实践
WebSocket 适合实时场景，加心跳检测。

### 面试回答示范
“HTTP 是单向，WebSocket 双向持久，适合聊天。我会加心跳保持连接。”

## 9. TCP 的拥塞控制是怎么实现的？

### 原理剖析
TCP 用慢启动、拥塞避免、快重传、快恢复控制流量，避免网络拥堵。

### 前端场景与最佳实践
下载大文件时分片请求，减轻压力。

### 面试回答示范
“TCP 拥塞控制用慢启动、拥塞避免等算法，丢包减窗口。前端下载大文件会分片优化。”

## 10. CDN 是什么？对前端有什么好处？

### 原理剖析
CDN 用分布式节点缓存内容，就近访问。

### 前端场景与最佳实践
加速静态资源，用版本号控缓存。

### 面试回答示范

## 11. HTTP/2 比 HTTP/1.1 有什么改进？

### 原理剖析
HTTP/2 基于 SPDY，改进：
1. **多路复用**：同一连接并行请求。
2. **头部压缩**：HPACK 减少开销。
3. **服务器推送**：主动推资源。

### 前端场景与最佳实践
用 HTTP/2 减少连接数，配合推送优化首屏。

### 面试回答示范
“HTTP/2 比 1.1 多了多路复用、头部压缩、服务器推送，减少延迟。前端用它优化加载，推送关键 CSS。”

## 12. TCP 四次挥手的过程是什么？

### 原理剖析
四次挥手断开连接：
1. 客户端发 FIN。
2. 服务器回 ACK。
3. 服务器发 FIN。
4. 客户端回 ACK。

确保双方都结束。

### 前端场景与最佳实践
长连接（如 WebSocket）断开时注意清理。

### 面试回答示范
“TCP 四次挥手是客户端发 FIN，服务器回 ACK，再发 FIN，客户端回 ACK，确保双方关闭。前端长连接断开要清理。”

## 13. IP 地址和 MAC 地址的区别是什么？

### 原理剖析
- **IP**：网络层，逻辑地址，跨网络路由。
- **MAC**：数据链路层，物理地址，局域网用。

### 前端场景与最佳实践
前端不直接操作，但理解有助于网络调试。

### 面试回答示范
“IP 是网络层地址，跨网路由；MAC 是物理地址，局域网用。前端不直接用，但调试网络时能理解数据流。”

## 14. OSI 七层模型和 TCP/IP 模型的区别？

### 原理剖析
- **OSI**：应用、表示、会话、传输、网络、数据链路、物理。
- **TCP/IP**：应用、传输、网际、链路，简化实用。

### 前端场景与最佳实践
理解模型定位问题（如 CORS 在应用层）。

### 面试回答示范
“OSI 七层从物理到应用，理论化；TCP/IP 四层更实用。前端关注应用层，比如 CORS。”

## 15. HTTP 的 Keep-Alive 是什么？

### 原理剖析
Keep-Alive 让 TCP 连接复用，避免频繁握手。

### 前端场景与最佳实践
默认开启，减少延迟。

### 面试回答示范
“Keep-Alive 让 HTTP 复用 TCP 连接，减少握手。前端默认用它优化性能。”

## 16. 什么是 DDoS 攻击？前端怎么防范？

### 原理剖析
DDoS（分布式拒绝服务）用大量请求淹没服务器。

### 前端场景与最佳实践
前端加验证码、限流，配合 CDN 防御。

### 面试回答示范
“DDoS 是用大量请求攻击服务器。前端加验证码、限流，配合 CDN 缓解。”

## 17. TCP 的粘包问题是什么？怎么解决？

### 原理剖析
TCP 是流式传输，数据可能粘在一起。解决：
1. 定长消息。
2. 分隔符。

### 前端场景与最佳实践
WebSocket 传输大数据时注意分包。

### 面试回答示范
“TCP 粘包是数据流没边界，可能粘连。解决用定长或分隔符。前端 WebSocket 传大数据会分包。”

## 19. 网络的 MTU 和 MSS 是什么？

### 原理剖析
- **MTU**：最大传输单元，链路层限制。
- **MSS**：最大分段大小，MTU 减头。

### 前端场景与最佳实践
大文件分片传输考虑 MSS。

### 面试回答示范
“MTU 是链路层最大单元，MSS 是 MTU 减头，TCP 用它分段。前端传大文件会分片考虑 MSS。”

## 20. QUIC 协议是什么？对前端有什么影响？

### 原理剖析
QUIC 基于 UDP，HTTP/3 用它，减少握手延迟，支持多路复用。

### 前端场景与最佳实践
未来用 HTTP/3 提升加载速度。

### 面试回答示范
“QUIC 基于 UDP，HTTP/3 用它，减少握手、多路复用。前端用 HTTP/3 会更快。”
