## 一、Promise 并发控制：别让请求挤爆浏览器 🚦

### 场景还原
面试官笑得一脸“慈祥”：“100 个 API 请求，浏览器最多跑 6 个，怎么控制？” 你要是甩 `Promise.all`，等着看浏览器“内存报警”吧💥。

### 原理剖析
并发控制是“限流”，像饭店点餐：6 个窗口，满了就排队。

#### 方案 1：手写调度器
```javascript
class PromisePool {
  constructor(max) {
    this.max = max;
    this.running = 0;
    this.queue = [];
  }

  add(task) {
    return new Promise((resolve, reject) => {
      const runner = () => {
        this.running++;
        task().then(resolve).catch(reject).finally(() => {
          this.running--;
          this.next();
        });
      };
      if (this.running < this.max) runner();
      else this.queue.push(runner);
    });
  }

  next() {
    if (this.queue.length && this.running < this.max) this.queue.shift()();
  }
}
```

#### 方案 2：async/await + 递归
```javascript
async function runWithLimit(tasks, limit) {
  const results = [];
  const executing = [];
  for (const task of tasks) {
    const p = Promise.resolve().then(() => task());
    results.push(p);
    if (limit <= tasks.length) {
      const e = p.then(() => executing.splice(executing.indexOf(e), 1));
      executing.push(e);
      if (executing.length >= limit) await Promise.race(executing);
    }
  }
  return Promise.all(results);
}
```

### 优缺点对比
- **手写调度器**：灵活，可加优先级；代码复杂。  
- **async/await**：简洁，原生；扩展性差。

### 应对策略
讲需求 → 画流程 → 写代码 → 提 p-limit。

**Tips**: 加 `timeout` 和 `retry`，超时跳过，失败重试。

## 三、React 渲染优化：别让组件“996”加班 🏃‍♂️

### 场景还原
“列表组件随父组件全重渲染，怎么优化？”

### 原理剖析
React 默认全量渲染，优化靠“精准更新”。

#### 方案 1：React.memo
```javascript
const ListItem = React.memo(({ item }) => <li>{item.name}</li>);
```

#### 方案 2：useMemo & useCallback
```javascript
function List({ data }) {
  const renderItem = useCallback(item => <li>{item.name}</li>, []);
  const items = useMemo(() => data.map(renderItem), [data]);
  return <ul>{items}</ul>;
}
```

### 优缺点对比
- **memo**：简单；对比函数易出错。  
- **useMemo/useCallback**：细粒度；依赖数组易漏。

### 应对策略
分析 → 用 memo 和 useCallback → 提 DevTools。

**Tips**: 大列表用 react-virtualized。

## 四、节流防抖：让事件“冷静”一下 ⏳

### 场景还原
“写防抖，和节流区别？啥场景用？”

### 原理剖析
- 防抖：等 N 秒，像等公交。  
- 节流：固定频率，像水龙头。

#### 实现
```javascript
function debounce(fn, delay) {
  let timer;
  return (...args) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}

function throttle(fn, interval) {
  let last = 0;
  return (...args) => {
    const now = Date.now();
    if (now - last >= interval) {
      fn(...args);
      last = now;
    }
  };
}
```

### 优缺点对比
- **防抖**：省资源；反馈慢。  
- **节流**：体验好；丢事件。

### 应对策略
讲场景 → 写代码 → 提 lodash。

**Tips**: 加 `leading/trailing` 参数。

## 十一、深浅拷贝：数据的“分身术” 🧪

### 场景还原
“写深拷贝，浅拷贝区别？”

### 原理剖析
- 浅拷贝：一层。  
- 深拷贝：全递归。

#### 实现
```javascript
function deepCopy(obj, cache = new WeakMap()) {
  if (typeof obj !== "object" || obj === null) return obj;
  if (cache.has(obj)) return cache.get(obj);
  const copy = Array.isArray(obj) ? [] : {};
  cache.set(obj, copy);
  for (let key in obj) {
    if (obj.hasOwnProperty(key)) copy[key] = deepCopy(obj[key], cache);
  }
  return copy;
}
```

### 优缺点对比
- **浅拷贝**：快；改引用崩。  
- **深拷贝**：独立；慢。

### 应对策略
讲区别 → 写深拷贝 → 提 JSON。

**Tips**: 用 WeakMap 防循环。

## 十五、数组方法优化：性能的“微雕大师” 🔧

### 场景还原
“forEach 和 map 哪个快？优化数组操作咋搞？”

### 原理剖析
数组方法性能差异源于实现和场景。

#### 示例
```javascript
const arr = Array(1000000).fill(1);
// forEach
console.time("forEach");
arr.forEach(x => x + 1);
console.timeEnd("forEach"); // 慢
// for
console.time("for");
for (let i = 0; i < arr.length; i++) arr[i] + 1;
console.timeEnd("for"); // 快
```

### 优缺点对比
- **forEach/map**：可读；慢。  
- **for**：快；可读差。

### 应对策略
讲性能 → 示例 → 提场景（大数据用 for）。

**Tips**: 用 reduce 替代多重循环。
