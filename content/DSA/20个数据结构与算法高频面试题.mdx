## 1. 反转链表
### 原理
链表反转就像把一串珍珠项链💍从尾到头重新串起来。单链表的每个节点只有指向下一个节点的指针，反转就是把这些指针“掉头”。

- **迭代法**：用 prev、curr、next 三个指针逐步翻转。
- **递归法**：从末尾回溯调整指针。

时间复杂度：O(n)，空间复杂度：迭代 O(1)，递归 O(n)。

### 最佳实践
- 检查空链表或单节点情况。
- 前端推荐迭代法，节省栈空间。
- 处理大数据时，考虑分块反转。

### 示例代码
```javascript
class ListNode {
  constructor(val = 0, next = null) {
    this.val = val;
    this.next = next;
  }
}

function reverseList(head) {
  let prev = null, curr = head;
  while (curr) {
    let next = curr.next;
    curr.next = prev;
    prev = curr;
    curr = next;
  }
  return prev;
}
```

### 面试示范答案
“面试官，反转链表可以用迭代或递归。我倾向于迭代法，用三个指针逐步翻转，时间复杂度 O(n)，空间复杂度 O(1)。比如从 1->2->3 变成 3->2->1。递归法更优雅，但栈空间是 O(n)，前端场景下我更推荐迭代法，比如处理 DOM 节点链表时更高效。您觉得还有什么需要注意的吗？”

## 2. 合并两个有序数组
### 原理
就像把两摞有序扑克牌🃏合并成一摞，我们要将两个已排序数组合并到 nums1 中。

- **双指针法**：从尾部比较，大的放后面。
- **额外空间法**：先复制再合并。

时间复杂度：O(m+n)，空间复杂度：O(1) 或 O(m)。

### 最佳实践
- nums1 空间足够时，从尾部合并最优。
- 前端场景：合并用户列表或时间线数据。

### 示例代码
```javascript
function merge(nums1, m, nums2, n) {
  let p1 = m - 1, p2 = n - 1, p = m + n - 1;
  while (p2 >= 0 && p1 >= 0) {
    nums1[p--] = nums1[p1] > nums2[p2] ? nums1[p1--] : nums2[p2--];
  }
  while (p2 >= 0) nums1[p--] = nums2[p2--];
}
```

### 面试示范答案
“我会用双指针从尾部合并，时间复杂度 O(m+n)，空间复杂度 O(1)。比如 nums1 = [1,2,3,0,0,0], nums2 = [2,5,6]，结果是 [1,2,2,3,5,6]。这种方法在前端合并有序数据时很实用，比如时间轴排序。如果空间不够，可以用额外数组，但空间复杂度会升到 O(m)。您觉得这个思路如何？”

## 3. 二叉树的中序遍历
### 原理
中序遍历就像逛超市🛒，先左货架（左子树），然后自己（根），再右货架（右子树）。

- **递归法**：直观。
- **迭代法**：用栈模拟。

时间复杂度：O(n)，空间复杂度：O(h)。

### 最佳实践
- 前端遍历 DOM 树常用中序。
- 迭代法更灵活，可中断。

### 示例代码
```javascript
class TreeNode {
  constructor(val = 0, left = null, right = null) {
    this.val = val;
    this.left = left;
    this.right = right;
  }
}

function inorderTraversal(root) {
  const result = [], stack = [];
  let curr = root;
  while (curr || stack.length) {
    while (curr) {
      stack.push(curr);
      curr = curr.left;
    }
    curr = stack.pop();
    result.push(curr.val);
    curr = curr.right;
  }
  return result;
}
```

### 面试示范答案
“中序遍历可以用递归或迭代。我选迭代法，用栈模拟，先把左子树入栈，弹出处理后再转向右子树。时间复杂度 O(n)，空间复杂度 O(h)。比如树 [1,null,2,3] 输出 [1,3,2]。前端解析 DOM 树时很实用。如果树很深，我会注意栈溢出问题。您觉得递归和迭代哪个更适合前端场景？”

## 4. 寻找两个正序数组的中位数
### 原理
就像找两队小朋友的中间高度🎒，我们要高效找到中位数。

- **暴力法**：合并后找中位数。
- **二分法**：找分割点。

时间复杂度：O(log(min(m,n)))。

### 最佳实践
- 用二分法，性能最优。
- 前端处理分页数据的中值分析。

### 示例代码
```javascript
function findMedianSortedArrays(nums1, nums2) {
  if (nums1.length > nums2.length) return findMedianSortedArrays(nums2, nums1);
  const m = nums1.length, n = nums2.length;
  let left = 0, right = m;
  while (left <= right) {
    const i = Math.floor((left + right) / 2);
    const j = Math.floor((m + n + 1) / 2) - i;
    const maxLeft1 = i === 0 ? -Infinity : nums1[i - 1];
    const minRight1 = i === m ? Infinity : nums1[i];
    const maxLeft2 = j === 0 ? -Infinity : nums2[j - 1];
    const minRight2 = j === n ? Infinity : nums2[j];
    if (maxLeft1 <= minRight2 && maxLeft2 <= minRight1) {
      const maxLeft = Math.max(maxLeft1, maxLeft2);
      const minRight = Math.min(minRight1, minRight2);
      return (m + n) % 2 ? maxLeft : (maxLeft + minRight) / 2;
    } else if (maxLeft1 > minRight2) right = i - 1;
    else left = i + 1;
  }
}
```

### 面试示范答案
“我会用二分法，时间复杂度 O(log(min(m,n)))。通过找分割点，确保左右元素数量相等且有序。比如 nums1 = [1,3], nums2 = [2]，中位数是 2。前端可以用在大数据集的中值计算。您觉得还有什么优化空间吗？”

## 5. 最长回文子串
### 原理
回文就像镜子里的倒影🪞，我们要找最长的回文子串。

- **中心扩展法**：从中心向两边扩展。

时间复杂度：O(n²)，空间复杂度：O(1)。

### 最佳实践
- 中心扩展法简单高效。
- 前端检查用户输入对称性。

### 示例代码
```javascript
function longestPalindrome(s) {
  let start = 0, maxLen = 1;
  function expandAroundCenter(left, right) {
    while (left >= 0 && right < s.length && s[left] === s[right]) {
      left--;
      right++;
    }
    return right - left - 1;
  }
  for (let i = 0; i < s.length; i++) {
    const len1 = expandAroundCenter(i, i);
    const len2 = expandAroundCenter(i, i + 1);
    const len = Math.max(len1, len2);
    if (len > maxLen) {
      start = i - Math.floor((len - 1) / 2);
      maxLen = len;
    }
  }
  return s.slice(start, start + maxLen);
}
```

### 面试示范答案
“我用中心扩展法，从每个字符向两边扩展，检查奇偶回文。时间复杂度 O(n²)，空间复杂度 O(1)。比如 'babad' 返回 'bab'。前端可以用在表单验证中。如果需要 O(n)，可以用 Manacher 算法。您觉得这个方法如何？”

## 6. LRU 缓存
### 原理
LRU 就像书桌📚，空间有限时移除最不常用的书。

- **实现**：哈希表 + 双向链表。

时间复杂度：O(1)。

### 最佳实践
- 用 Map 简化实现。
- 前端场景：浏览器缓存。

### 示例代码
```javascript
class LRUCache {
  constructor(capacity) {
    this.capacity = capacity;
    this.cache = new Map();
  }
  get(key) {
    if (!this.cache.has(key)) return -1;
    const value = this.cache.get(key);
    this.cache.delete(key);
    this.cache.set(key, value);
    return value;
  }
  put(key, value) {
    if (this.cache.has(key)) this.cache.delete(key);
    else if (this.cache.size >= this.capacity) {
      const firstKey = this.cache.keys().next().value;
      this.cache.delete(firstKey);
    }
    this.cache.set(key, value);
  }
}
```

### 面试示范答案
“我用 Map 实现 LRU，get 和 put 都是 O(1)。比如 capacity = 2，put(1,1), put(2,2), get(1), put(3,3)，缓存变成 [1,3]。前端可以用在图片缓存中。您觉得 Map 有没有什么潜在问题？”

## 7. 岛屿数量
### 原理
地图上🌍，1 是陆地，0 是水，求独立岛屿数。

- **DFS**：淹没相连陆地。

时间复杂度：O(m*n)。

### 最佳实践
- DFS 直观，注意栈空间。
- 前端分析网格连通性。

### 示例代码
```javascript
function numIslands(grid) {
  let count = 0;
  function dfs(i, j) {
    if (i < 0 || i >= grid.length || j < 0 || j >= grid[0].length || grid[i][j] === "0") return;
    grid[i][j] = "0";
    dfs(i + 1, j);
    dfs(i - 1, j);
    dfs(i, j + 1);
    dfs(i, j - 1);
  }
  for (let i = 0; i < grid.length; i++) {
    for (let j = 0; j < grid[0].length; j++) {
      if (grid[i][j] === "1") {
        count++;
        dfs(i, j);
      }
    }
  }
  return count;
}
```

### 面试示范答案
“我用 DFS，遇到 '1' 就淹没相连陆地，时间复杂度 O(m*n)。比如 [['1','1'],['0','1']] 返回 1。前端可以用在网格布局分析中。如果网格很大，可以试试 BFS。您觉得 DFS 和 BFS 哪个更合适？”

## 8. 滑动窗口最大值
### 原理
滑动窗口就像火车窗🚂，找每个窗口的最大值。

- **单调队列**：维护递减序列。

时间复杂度：O(n)。

### 最佳实践
- 单调队列高效。
- 前端实时数据分析。

### 示例代码
```javascript
function maxSlidingWindow(nums, k) {
  const result = [], deque = [];
  for (let i = 0; i < nums.length; i++) {
    while (deque.length && nums[deque[deque.length - 1]] <= nums[i]) deque.pop();
    deque.push(i);
    if (deque[0] <= i - k) deque.shift();
    if (i >= k - 1) result.push(nums[deque[0]]);
  }
  return result;
}
```

### 面试示范答案
“我用单调队列，维护递减序列，队首是最大值。时间复杂度 O(n)。比如 [1,3,-1,-3,5,3,6,7], k = 3，返回 [3,3,5,5,6,7]。前端可以用在滚动数据监控中。您觉得有什么改进空间吗？”

## 9. 字符串解码
### 原理
就像解压文件📦，把 "3[a]2[bc]" 解成 "aaabcbc"。

- **栈法**：存数字和字符。

时间复杂度：O(n)。

### 最佳实践
- 栈处理嵌套清晰。
- 前端解析模板。

### 示例代码
```javascript
function decodeString(s) {
  const numStack = [], strStack = [];
  let num = 0, str = '';
  for (const char of s) {
    if (/\d/.test(char)) num = num * 10 + Number(char);
    else if (char === '[') {
      numStack.push(num);
      strStack.push(str);
      num = 0;
      str = '';
    } else if (char === ']') {
      const times = numStack.pop();
      str = strStack.pop() + str.repeat(times);
    } else str += char;
  }
  return str;
}
```

### 面试示范答案
“我用两个栈，数字栈和字符串栈，遇到 ']' 时弹出重复。时间复杂度 O(n)。比如 '3[a]2[bc]' 返回 'aaabcbc'。前端可以用在模板渲染中。您觉得有什么应用场景吗？”

## 10. 最小覆盖子串
### 原理
就像找最小的窗帘🎪，覆盖所有目标字符。

- **滑动窗口**：双指针 + 哈希表。

时间复杂度：O(n)。

### 最佳实践
- 哈希表记录频率。
- 前端搜索关键词。

### 示例代码
```javascript
function minWindow(s, t) {
  const map = new Map();
  for (const char of t) map.set(char, (map.get(char) || 0) + 1);
  let left = 0, count = t.length, minLen = Infinity, minStart = 0;
  for (let right = 0; right < s.length; right++) {
    if (map.has(s[right])) {
      map.set(s[right], map.get(s[right]) - 1);
      if (map.get(s[right]) >= 0) count--;
    }
    while (count === 0) {
      if (right - left + 1 < minLen) {
        minLen = right - left + 1;
        minStart = left;
      }
      if (map.has(s[left])) {
        map.set(s[left], map.get(s[left]) + 1);
        if (map.get(s[left]) > 0) count++;
      }
      left++;
    }
  }
  return minLen === Infinity ? '' : s.slice(minStart, minStart + minLen);
}
```

### 面试示范答案
“我用滑动窗口，Map 记录频率，right 扩展，left 收缩。时间复杂度 O(n)。比如 s = 'ADOBECODEBANC', t = 'ABC'，返回 'BANC'。前端可以用在搜索高亮中。您觉得有什么优化空间吗？”

## 11. 两数之和
### 原理
就像在人群中找两个搭档👯，他们的和是目标值。

- **哈希表法**：存差值。

时间复杂度：O(n)。

### 最佳实践
- 用 Map 一次遍历。
- 前端匹配数据对。

### 示例代码
```javascript
function twoSum(nums, target) {
  const map = new Map();
  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];
    if (map.has(complement)) return [map.get(complement), i];
    map.set(nums[i], i);
  }
  return [];
}
```

### 面试示范答案

## 12. 有效的括号
### 原理
括号匹配就像叠盘子🍽️，先进后出。

时间复杂度：O(n)。

### 最佳实践
- 用栈检查配对。

### 示例代码
```javascript
function isValid(s) {
  const stack = [];
  const map = { ')': '(', '}': '{', ']': '[' };
  for (const char of s) {
    if (!map[char]) stack.push(char);
    else if (stack.pop() !== map[char]) return false;

}
```

## 13. 二叉树的层序遍历
### 原理
层序遍历就像参观大楼🏢，一层一层看。

- **BFS**：用队列。

时间复杂度：O(n)。

### 最佳实践
- 用队列实现。
- 前端渲染树形结构。

### 示例代码
```javascript
function levelOrder(root) {
  if (!root) return [];
  const result = [], queue = [root];
  while (queue.length) {
    const level = [];
    const size = queue.length;
    for (let i = 0; i < size; i++) {
      const node = queue.shift();
      level.push(node.val);
      if (node.left) queue.push(node.left);
      if (node.right) queue.push(node.right);
    }
    result.push(level);
  }
  return result;
}
```

### 面试示范答案
“我用队列实现层序遍历，逐层处理。时间复杂度 O(n)。比如树 [3,9,20,null,null,15,7] 返回 [[3],[9,20],[15,7]]。前端可以用在树形组件渲染中。您觉得有什么优化吗？”

```javascript
function maxSubArray(nums) {
```

## 15. 合并区间
### 原理
就像整理日程表📅，合并重叠区间。

- **排序法**：先排序再合并。

时间复杂度：O(n log n)。

### 最佳实践
- 按起始点排序。
- 前端处理时间轴。

### 示例代码
```javascript
function merge(intervals) {
  intervals.sort((a, b) => a[0] - b[0]);
  const result = [intervals[0]];
  for (let i = 1; i < intervals.length; i++) {
    const last = result[result.length - 1];
    if (intervals[i][0] <= last[1]) last[1] = Math.max(last[1], intervals[i][1]);
    else result.push(intervals[i]);
  }
  return result;
}
```

### 面试示范答案
“我先按起始点排序，然后逐个合并。时间复杂度 O(n log n)。比如 [[1,3],[2,6],[8,10]] 返回 [[1,6],[8,10]]。前端可以用在时间轴优化中。您觉得有什么改进吗？”

## 16. 接雨水
### 原理
就像计算屋顶积水🏠，找每个柱子能接的水量。

- **双指针法**：左右指针夹逼。

时间复杂度：O(n)。

### 最佳实践
- 双指针高效。
- 前端可视化积水。

### 示例代码
```javascript
function trap(height) {
  let left = 0, right = height.length - 1, leftMax = 0, rightMax = 0, water = 0;
  while (left < right) {
    leftMax = Math.max(leftMax, height[left]);
    rightMax = Math.max(rightMax, height[right]);
    if (leftMax < rightMax) water += leftMax - height[left++];
    else water += rightMax - height[right--];
  }
  return water;
}
```

### 面试示范答案
“我用双指针，从两端夹逼，计算每个位置的积水。时间复杂度 O(n)。比如 [0,1,0,2,1,0,1,3,2,1,2,1] 返回 6。前端可以用在柱状图分析中。您觉得有什么要注意的吗？”

## 17. 环形链表
### 原理
就像跑道是否有环🏃，检测链表是否有循环。

- **快慢指针**：快指针追上慢指针。

时间复杂度：O(n)。

### 最佳实践
- 快慢指针简单。
- 前端检测循环引用。

### 示例代码
```javascript
function hasCycle(head) {
  let slow = head, fast = head;
  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow === fast) return true;
  }
  return false;
}
```

### 面试示范答案
“我用快慢指针，快指针每次走两步，慢指针走一步，相遇说明有环。时间复杂度 O(n)。比如 1->2->3->2，返回 true。前端可以用在检测对象循环引用中。您觉得还有什么方法吗？”

## 18. 前 K 个高频元素
### 原理
就像找热门歌曲🎵，求出现频率前 K 的元素。

- **堆法**：用小顶堆。

时间复杂度：O(n log k)。

### 最佳实践
- 用 Map + 堆。
- 前端统计热词。

### 示例代码
```javascript
function topKFrequent(nums, k) {
  const map = new Map();
  for (const num of nums) map.set(num, (map.get(num) || 0) + 1);
  return [...map.entries()]
    .sort((a, b) => b[1] - a[1])
    .slice(0, k)
    .map(([num]) => num);
}
```

### 面试示范答案
“我用 Map 统计频率，再排序取前 K。时间复杂度 O(n log n)，可以用小顶堆优化到 O(n log k)。比如 [1,1,1,2,2,3], k = 2，返回 [1,2]。前端可以用在搜索热词统计中。您觉得堆实现怎么样？”

## 19. 课程表（拓扑排序）
### 原理
就像安排课程表📚，检测是否有环并排序。

- **DFS**：检测环。
- **BFS**：入度为 0 入队。

时间复杂度：O(V + E)。

### 最佳实践
- BFS 更直观。
- 前端依赖管理。

### 示例代码
```javascript
function canFinish(numCourses, prerequisites) {
  const graph = Array(numCourses).fill().map(() => []);
  const inDegree = Array(numCourses).fill(0);
  for (const [course, pre] of prerequisites) {
    graph[pre].push(course);
    inDegree[course]++;
  }
  const queue = [];
  for (let i = 0; i < numCourses; i++) if (inDegree[i] === 0) queue.push(i);
  let count = 0;
  while (queue.length) {
    const curr = queue.shift();
    count++;
    for (const next of graph[curr]) {
      inDegree[next]--;
      if (inDegree[next] === 0) queue.push(next);
    }
  }
  return count === numCourses;
}
```

### 面试示范答案
“我用 BFS 实现拓扑排序，建图并计算入度，入度为 0 的课程入队。时间复杂度 O(V + E)。比如 [ [1,0] ] 返回 true。前端可以用在依赖解析中。您觉得 DFS 怎么样？”

## 20. 跳跃游戏
### 原理
就像跳格子游戏🎲，判断能否跳到最后。

- **贪心法**：维护最远距离。

时间复杂度：O(n)。

### 最佳实践
- 贪心简单高效。
- 前端游戏逻辑。

### 示例代码
```javascript
function canJump(nums) {
  let maxReach = 0;
  for (let i = 0; i < nums.length; i++) {
    if (i > maxReach) return false;
    maxReach = Math.max(maxReach, i + nums[i]);
    if (maxReach >= nums.length - 1) return true;
  }
  return true;
}
```

### 面试示范答案
“我用贪心法，维护能跳到的最远距离。时间复杂度 O(n)。比如 [2,3,1,1,4] 返回 true。前端可以用在游戏跳跃逻辑中。您觉得有什么其他方法吗？”
