## 1. 什么是语义化 HTML？为什么要用？

### 原理剖析
语义化 HTML 是指使用具有明确意义的标签（如 `<header>`、`<article>`、`<footer>`）来描述内容，而不是单纯依赖 `<div>` 和 `<span>`。它的核心在于让代码的结构反映内容的含义。比如，`<h1>` 告诉浏览器和搜索引擎“这是一个标题”，而 `<section>` 表示“这里是一个独立的内容块”。

浏览器解析 HTML 时会构建 DOM 树，语义化标签能让 DOM 结构更清晰。更重要的是，屏幕阅读器等辅助技术会依赖这些标签来理解页面结构，提升可访问性（Accessibility，简称 A11y）。

### 最佳实践
- 用 `<nav>` 包裹导航栏，而不是 `<div class="nav">`。
- 用 `<main>` 表示主要内容区域，辅助技术会优先聚焦这里。
- 避免滥用 `<div>`，比如用 `<article>` 表示博客文章，用 `<aside>` 表示侧边栏。
- 配合 ARIA 属性（如 `role` 和 `aria-label`）进一步增强语义。

### 示例
```html
<!-- 非语义化 -->
<div class="header">
  <div class="title">我的博客</div>
</div>

<!-- 语义化 -->
<header>
  <h1>我的博客</h1>
</header>
```

### 面试官示范答案
“语义化 HTML 是指用合适的标签表达内容的含义，比如用 `<header>` 表示页头，`<footer>` 表示页脚。它的好处有三点：一是提高代码可读性，团队协作更高效；二是对 SEO 友好，搜索引擎能更好地理解页面结构；三是提升可访问性，屏幕阅读器能准确识别内容。我在项目中会优先使用语义化标签，并结合 ARIA 属性，确保代码既优雅又有用。”

## 2. DOCTYPE 有什么作用？不写会怎样？

### 原理剖析
`<!DOCTYPE>` 是文档类型声明，告诉浏览器当前 HTML 文档的版本和解析规则。比如 `<!DOCTYPE html>` 表示 HTML5，浏览器会以标准模式（Standards Mode）渲染页面。如果不写，浏览器会进入怪异模式（Quirks Mode），模仿早期浏览器的行为，导致 CSS 盒模型等计算方式出现偏差。

怪异模式是为了兼容古老网页设计的“历史包袱”，但现代开发中几乎没人会用它。

### 最佳实践
- 始终在 HTML 文件顶部写上 `<!DOCTYPE html>`。
- 检查团队模板是否默认包含 DOCTYPE，避免遗漏。
- 如果接手老项目，确认 DOCTYPE，避免调试时踩坑。

### 示例
```html
<!-- 标准模式 -->
<!DOCTYPE html>
<html>
  <body>
    <div style="width: 100px; height: 100px; border: 10px solid;">盒子</div>
  </body>
</html>

<!-- 怪异模式（无 DOCTYPE） -->
<html>
  <body>
    <div style="width: 100px; height: 100px; border: 10px solid;">盒子</div>
  </body>
</html>
```
在怪异模式下，盒子宽高可能是 120px（包括边框），而在标准模式下是 100px（边框外扩）。

### 面试官示范答案
“`<!DOCTYPE>` 是文档类型声明，用来指定 HTML 版本，比如 `<!DOCTYPE html>` 表示 HTML5。它的作用是让浏览器以标准模式解析页面，避免进入怪异模式。如果不写，浏览器会用怪异模式渲染，CSS 盒模型可能会出错，比如边框计算方式不同。我在项目中会确保每个页面都加上 DOCTYPE，保证一致性和兼容性。”

## 3. HTML5 新增了哪些特性？

### 原理剖析
HTML5 是对 HTML4 的全面升级，引入了新标签、新 API 和多媒体支持。核心目标是让 Web 更现代化，支持动态内容和移动端开发。新增特性包括：
- **语义化标签**：`<section>`、`<article>` 等。
- **多媒体**：`<audio>` 和 `<video>`，无需 Flash。
- **表单增强**：`type="email"`、`placeholder` 等。
- **API**：Canvas、Web Storage、Geolocation 等。

这些特性减少了对插件的依赖，提升了性能和用户体验。

### 最佳实践
- 用 `<video>` 替代 Flash 播放器，支持多种格式（如 MP4、WebM）。
- 用 `localStorage` 存储小型数据，替代 Cookie。
- 用 `<input type="date">` 提供原生日期选择器，提升移动端体验。

### 示例
```html
<video controls>
  <source src="movie.mp4" type="video/mp4">
  您的浏览器不支持视频播放。
</video>
```

### 面试官示范答案
“HTML5 新增了很多特性，比如语义化标签（像 `<header>`、`<footer>`），让结构更清晰；多媒体标签 `<audio>` 和 `<video>`，摆脱了 Flash；还有表单增强，像 `type="email"` 和 `required`，提升用户体验。此外还有 Canvas、Web Storage 等 API，支持更复杂的应用。我在项目中常用 `<video>` 处理视频，用 `localStorage` 缓存数据，效果很好。”

## 4. 如何优化 HTML 页面的加载速度？

### 原理剖析
HTML 页面加载涉及解析 DOM、加载资源和渲染三个阶段。优化加载速度的核心是减少阻塞、压缩资源和提前加载。浏览器会从上到下解析 HTML，遇到外部资源（如 CSS、JS）可能阻塞渲染。

### 最佳实践
- **异步加载**：`<script async>` 或 `<script defer>` 避免阻塞。
- **预加载**：用 `<link rel="preload">` 提前加载关键资源。
- **压缩 HTML**：去除多余空格和注释。
- **CDN**：通过内容分发网络加速静态资源加载。
- **图片优化**：用 `<img loading="lazy">` 延迟加载非视口图片。

### 示例
```html
<head>
  <link rel="preload" href="style.css" as="style">
</head>
<body>
  <img src="hero.jpg" loading="lazy" alt="英雄图">
  <script src="app.js" defer></script>
</body>
```

### 面试官示范答案
“优化 HTML 加载速度可以从几方面入手：一是用 `async` 或 `defer` 异步加载 JS，避免阻塞；二是用 `<link rel="preload">` 预加载关键资源；三是压缩 HTML 和用 CDN 加速资源加载。我还喜欢用 `loading="lazy"` 延迟加载图片，特别适合长页面，提升首屏速度。实际项目中我会结合工具像 Lighthouse 验证效果。”

## 5. `<meta>` 标签有哪些常用场景？

### 原理剖析
`<meta>` 标签提供页面的元数据，影响浏览器行为、SEO 和移动端适配。它不直接显示，但对页面功能至关重要。常见属性包括 `charset`、`name` 和 `content`。

### 最佳实践
- **字符编码**：`<meta charset="UTF-8">` 防止乱码。
- **视口控制**：`<meta name="viewport" content="width=device-width, initial-scale=1.0">` 适配移动端。
- **SEO**：`<meta name="description" content="页面描述">` 提升搜索排名。
- **防缓存**：`<meta http-equiv="cache-control" content="no-cache">`。

### 示例
```html
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="description" content="前端开发者的 HTML 面试指南">
```

### 面试官示范答案
“`<meta>` 标签用来定义页面元数据，常用场景包括设置 `charset="UTF-8"` 确保编码正确，用 `viewport` 适配移动端，还有用 `description` 优化 SEO。我在项目中还会用 `http-equiv` 控制缓存，比如强制刷新。合理使用 `<meta>` 能提升页面兼容性和用户体验。”

## 6. 如何实现一个可访问性强的表单？

### 原理剖析
可访问性强的表单意味着屏幕阅读器用户和键盘用户也能轻松操作。核心在于提供足够的上下文（如标签）和导航支持。浏览器和辅助技术会解析 `<label>`、ARIA 属性等。

### 最佳实践
- 用 `<label>` 关联输入框，点击标签也能聚焦。
- 添加 `aria-label` 或 `aria-describedby` 补充说明。
- 确保表单元素有 `:focus` 样式，支持键盘导航。
- 用 `<fieldset>` 和 `<legend>` 分组相关字段。

### 示例
```html
<form>
  <fieldset>
    <legend>个人信息</legend>
    <label for="name">姓名</label>
    <input id="name" type="text" aria-required="true">
  </fieldset>
</form>
```

### 面试官示范答案
“要实现可访问性强的表单，我会用 `<label>` 关联输入框，确保点击标签能聚焦；用 ARIA 属性像 `aria-required` 提示必填项；还会在 CSS 中加 `:focus` 样式支持键盘导航。项目中我会用工具像 axe 测试，确保表单对所有用户友好。”

## 7. `<iframe>` 有哪些优缺点？

### 原理剖析
`<iframe>` 嵌入外部内容（如视频、地图），本质上是一个独立的浏览器窗口。它有自己的 DOM 和上下文，与父页面隔离。

### 优点
- 隔离性强，适合嵌入第三方内容。
- 支持动态加载，比如嵌入 YouTube 视频。

### 缺点
- 性能开销大，多个 iframe 可能拖慢页面。
- SEO 不友好，搜索引擎难以索引内容。
- 移动端适配麻烦，可能出现滚动问题。

### 最佳实践
- 尽量用原生替代（如 `<video>` 替代视频 iframe）。
- 用 `loading="lazy"` 延迟加载非关键 iframe。

### 面试官示范答案
“`<iframe>` 的优点是隔离性好，适合嵌入第三方内容，比如地图或视频；缺点是性能开销大，对 SEO 不利。我在项目中会优先用原生标签替代，如果必须用 iframe，会加 `loading="lazy"` 优化加载。”

## 8. HTML 中的 data 属性有什么用？

### 原理剖析
`data-*` 是 HTML5 引入的自定义属性，用于存储私有数据。浏览器不会直接处理，但 JS 可以通过 `dataset` API 访问。

### 最佳实践
- 用 `data-id` 标记元素，方便 JS 操作。
- 避免用作样式钩子，交给 CSS 类。
- 命名清晰，如 `data-user-id` 而不是 `data-uid`。

### 示例
```html
<div data-product-id="123" data-category="book">书籍</div>
<script>
  const div = document.querySelector('div');
  console.log(div.dataset.productId); // "123"
</script>
```

### 面试官示范答案
“`data-*` 属性用来存储自定义数据，比如 `data-id` 标记元素，JS 通过 `dataset` 访问。它很适合动态交互，我在项目中常用它绑定数据，但不会用作样式钩子，保持职责分离。”

## 9. 如何处理浏览器兼容性？

### 原理剖析
不同浏览器对 HTML 和 CSS 的支持程度不同（如 IE 不支持某些 HTML5 标签）。兼容性处理需要检测特性并提供回退方案。

### 最佳实践
- 用 Modernizr 检测特性支持。
- 用 `<picture>` 提供多格式图片。
- 加 vendor 前缀（如 `-webkit-`）支持老浏览器。

### 示例
```html
<picture>
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="图片">
</picture>
```

### 面试官示范答案
“处理浏览器兼容性，我会用 Modernizr 检测特性支持，比如 HTML5 标签；用 `<picture>` 提供图片回退；必要时加 vendor 前缀。项目中我会用 caniuse.com 检查支持情况，确保代码健壮。”

## 10. 如何防止 XSS 攻击？

### 原理剖析
XSS（跨站脚本攻击）是通过注入恶意脚本运行在用户浏览器中。HTML 层面可以通过编码和限制输入防止。

### 最佳实践
- 用 `textContent` 而不是 `innerHTML` 设置文本。
- 对用户输入进行转义（如 `<` 转为 `<`）。
- 用 CSP（内容安全策略）限制脚本来源。

### 示例
```html
<script>
  const userInput = '<script>alert("hack")</script>';
  document.body.textContent = userInput; // 安全
</script>
```

### 面试官示范答案
“防止 XSS，我会用 `textContent` 替代 `innerHTML` 处理用户输入，避免脚本执行；对输入进行转义；还会用 CSP 限制外部脚本。项目中我会配合后端验证，确保安全。”

## 11. `<link>` 标签有哪些用途？

### 原理剖析
`<link>` 标签主要用于引入外部资源，最常见的是加载 CSS 文件。它通过 `rel` 属性定义与资源的关系，比如 `stylesheet` 表示样式表，`icon` 表示网站图标。浏览器会根据 `rel` 值决定如何处理资源，比如 `preload` 会提前加载但不执行。

### 最佳实践
- 用 `<link rel="stylesheet">` 加载 CSS，确保样式优先级清晰。
- 用 `<link rel="preload">` 优化关键资源加载，比如字体。
- 用 `<link rel="icon">` 设置 favicon，提升品牌识别度。
- 避免用过多的 `<link>` 阻塞渲染，必要时合并资源。

### 示例
```html
<link rel="stylesheet" href="styles.css">
<link rel="preload" href="font.woff2" as="font" type="font/woff2">
<link rel="icon" href="favicon.ico" type="image/x-icon">
```

### 面试官示范答案
“`<link>` 标签用来引入外部资源，最常见是 `<link rel="stylesheet">` 加载 CSS。我还喜欢用 `rel="preload"` 提前加载字体或关键脚本，提升性能；用 `rel="icon"` 设置网站图标。项目中我会注意资源数量，尽量合并 CSS，避免阻塞渲染。”

## 12. HTML 中的 `defer` 和 `async` 有什么区别？

### 原理剖析
`<script>` 标签默认是同步加载，会阻塞 HTML 解析。`defer` 和 `async` 是 HTML5 引入的属性，用来优化脚本加载。`defer` 表示脚本延迟到 HTML 解析完成后执行，且按顺序执行；`async` 表示脚本异步加载，下载完立即执行，不保证顺序。

### 最佳实践
- 用 `defer` 加载依赖 DOM 的脚本，比如操作 DOM 的库。
- 用 `async` 加载独立脚本，比如埋点统计。
- 避免混用，确保脚本依赖关系清晰。

### 示例
```html
<script src="jquery.js" defer></script>
<script src="analytics.js" async></script>
```

### 面试官示范答案
“`defer` 和 `async` 都是优化脚本加载的属性。`defer` 让脚本在 HTML 解析完后再按顺序执行，适合依赖 DOM 的代码；`async` 是下载完就执行，不保证顺序，适合独立脚本。我在项目中会用 `defer` 加载 jQuery，用 `async` 加载统计脚本，保证性能和功能。”

## 13. 什么是 HTML 的实体编码？为什么要用？

### 原理剖析
HTML 实体编码是用特殊字符（如 `&lt;` 表示 `<`）替代原始字符，避免浏览器误解析。比如，直接写 `<` 会被当成标签开始，而 `&lt;` 会被渲染成 `<` 符号。实体编码还能防止 XSS 攻击，提升安全性。

### 最佳实践
- 对用户输入一律编码，防止注入。
- 用 `&amp;` 表示 `&`，确保链接参数正确。
- 熟悉常用实体，如 `&nbsp;`（不换行空格）、`&quot;`（引号）。

### 示例
```html
<p>5 &lt; 10</p> <!-- 显示为 5 < 10 -->
<a href="page.html?x=1&amp;y=2">链接</a>
```

### 面试官示范答案
“HTML 实体编码是用特殊字符表示原始字符，比如 `&lt;` 表示 `<`。它能防止浏览器误解析标签，还能防御 XSS。我在项目中会对用户输入自动编码，比如用 `&amp;` 处理 URL 参数，确保安全和正确性。”

## 14. `<canvas>` 有什么用途？如何优化性能？

### 原理剖析
`<canvas>` 是 HTML5 提供的绘图标签，通过 JS 操作 2D 或 WebGL 上下文绘制图形。它的本质是一个位图画布，适合动态渲染，比如游戏或数据可视化。但频繁重绘可能导致性能瓶颈。

### 最佳实践
- 用 `requestAnimationFrame` 替代 `setInterval` 优化动画。
- 离屏渲染（`OffscreenCanvas`）处理复杂图形。
- 减少绘制区域，只更新变化部分。

### 示例
```html
<canvas id="myCanvas" width="400" height="400"></canvas>
<script>
  const canvas = document.getElementById('myCanvas');
  const ctx = canvas.getContext('2d');
  function draw() {
    ctx.clearRect(0, 0, 400, 400);
    ctx.fillRect(50, 50, 100, 100);
    requestAnimationFrame(draw);
  }
  draw();
</script>
```

### 面试官示范答案
“`<canvas>` 用来绘制动态图形，比如图表或游戏，通过 2D 或 WebGL 上下文操作。我会用 `requestAnimationFrame` 优化动画性能，必要时用离屏渲染减少开销。项目中我会尽量只重绘变化区域，提升效率。”

## 15. 如何用 HTML 实现懒加载？

### 原理剖析
懒加载（Lazy Loading）是延迟加载非关键资源的技术，减少首屏加载时间。HTML5 引入了 `loading="lazy"` 属性，适用于 `<img>` 和 `<iframe>`，浏览器会在元素接近视口时自动加载。

### 最佳实践
- 对视口外的 `<img>` 加 `loading="lazy"`。
- 配合 `IntersectionObserver` 实现自定义懒加载。
- 提供 `width` 和 `height` 属性，防止布局偏移。

### 示例
```html
<img src="photo.jpg" loading="lazy" width="300" height="200" alt="图片">
```

### 面试官示范答案
“HTML 实现懒加载可以用 `loading="lazy"`，它让浏览器自动延迟加载图片或 iframe，很适合长页面。我还会加 `width` 和 `height` 防止跳动。如果需要更灵活的控制，我会用 `IntersectionObserver` 写自定义逻辑。”

## 16. `<template>` 标签有什么作用？

### 原理剖析
`<template>` 是 HTML5 的一个容器标签，里面的内容不会直接渲染，但可以通过 JS 克隆使用。它就像一个“模具”，适合动态生成重复结构。

### 最佳实践
- 用 `<template>` 存储列表项模板。
- 结合 `document.importNode` 高效克隆内容。
- 避免直接操作 `<template>` 的 DOM，保持其纯净。

### 示例
```html
<template id="item-template">
  <li>项目</li>
</template>
<script>
  const template = document.getElementById('item-template');
  const clone = document.importNode(template.content, true);
  document.querySelector('ul').appendChild(clone);
</script>
```

### 面试官示范答案
“`<template>` 是个不渲染的容器，适合存模板代码，比如列表项。我在项目中会用它配合 JS 动态生成内容，用 `importNode` 克隆，既高效又清晰。它就像个模具，能省不少重复代码。”

## 17. HTML 中的 `contenteditable` 属性有什么用？

### 原理剖析
`contenteditable` 是一个全局属性，设置为 `true` 时，元素内容可编辑。它本质上是浏览器提供的原生富文本功能，常用于简单编辑器。

### 最佳实践
- 用在 `<div>` 或 `<p>` 上实现可编辑区域。
- 配合 `input` 事件监听实时变化。
- 注意安全性，避免 XSS（不要直接用 `innerHTML` 输出）。

### 示例
```html
<div contenteditable="true">点击编辑我</div>
<script>
  document.querySelector('div').addEventListener('input', (e) => {
    console.log(e.target.textContent);
  });
</script>
```

### 面试官示范答案
“`contenteditable` 让元素可编辑，比如实现一个简单的在线编辑器。我会用它在 `<div>` 上做内容输入区，监听 `input` 事件处理变化。项目中我会注意转义输出，防止 XSS 风险。”

## 18. 如何用 HTML 实现响应式图片？

### 原理剖析
响应式图片根据设备分辨率或视口大小加载不同版本的图片。HTML 提供了 `<picture>` 和 `srcset` 两种方案，前者更灵活，后者更简洁。

### 最佳实践
- 用 `<picture>` 提供多种格式（如 WebP 和 JPG）。
- 用 `srcset` 和 `sizes` 适配 DPR（设备像素比）。
- 确保回退 `<img>` 有 `alt` 属性。

### 示例
```html
<picture>
  <source media="(min-width: 768px)" srcset="large.webp" type="image/webp">
  <img src="small.jpg" alt="响应式图片">
</picture>
```

### 面试官示范答案
“响应式图片可以用 `<picture>` 或 `srcset` 实现。`<picture>` 适合根据条件加载不同图片，比如大屏用 WebP，小屏用 JPG；`srcset` 适合适配像素密度。我会确保有回退方案，提升性能和兼容性。”

## 19. HTML 中的 `rel` 属性有哪些值？

### 原理剖析
`rel` 属性定义当前元素与链接资源的关系，常用于 `<a>` 和 `<link>`。常见值包括 `nofollow`（不追踪）、`noopener`（防止窗口操作）、`noreferrer`（不发送引用头）。

### 最佳实践
- 用 `rel="nofollow"` 标记不可信链接，优化 SEO。
- 用 `rel="noopener"` 防止新窗口操作父页面。
- 用 `rel="alternate"` 标记 RSS 订阅。

### 示例
```html
<a href="https://example.com" rel="nofollow noopener" target="_blank">外部链接</a>
```

### 面试官示范答案
“`rel` 属性定义链接关系，比如 `nofollow` 告诉搜索引擎不追踪，`noopener` 防止新窗口操作父页面。我在项目中会给外部链接加 `rel="noopener nofollow"`，既安全又对 SEO 友好。”

## 20. 如何用 HTML 实现打印优化？

### 原理剖析
打印优化通过 CSS 媒体查询（`@media print`）和 HTML 属性控制打印样式。浏览器在打印时会应用特定规则，去掉不必要的元素。

### 最佳实践
- 用 `<link media="print">` 加载打印专用 CSS。
- 加 `print-only` 类隐藏屏幕内容。
- 用 `<pagebreak>` 控制分页。

### 示例
```html
<link rel="stylesheet" href="print.css" media="print">
<style>
  @media print {
    .no-print { display: none; }
  }
</style>
<div class="no-print">仅屏幕显示</div>
```

### 面试官示范答案
“打印优化可以用 `<link media="print">` 加载专用样式，或者用 `@media print` 隐藏不需要的元素。我在项目中会加 `no-print` 类去掉导航栏，确保打印内容简洁清晰。”
